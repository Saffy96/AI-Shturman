# Frontend: Hybrid Premium Map redesign

## Аудит исходной архитектуры

До редизайна `App.tsx` одновременно управлял доменными данными, запросами, состоянием маршрута и композицией интерфейса. Полноэкранная карта уже существовала внутри `CarShell`, однако основной пользовательский поток оставался длинной прокручиваемой колонкой в универсальном `BottomSheet`: режим поиска, форма маршрута, фильтры, сводка и весь список АЗС находились в одном визуальном контейнере.

Карта реализована в `components/Map.tsx`. В ней уже были сохранение zoom, различение программных и ручных перемещений, одноразовый `fitBounds` при смене маршрута, memo-маркеры и throttled сохранение масштаба. Маршрут и поиск используют состояние и обработчики `App.tsx`; API-контракты сосредоточены в `services/fuelApi.ts`. Детали станции лениво загружаются компонентом `FuelStationCard` только после выбора/раскрытия и кэшируются в локальном состоянии карточки. Эти части оставлены без изменения контрактов.

## Новая композиция

`App.tsx` теперь является компактным composition root. Доменное состояние, запросы и сценарии поиска сосредоточены в `features/navigation/useNavigatorController.ts`, persistent-настройки — в `features/navigation/storage.ts`, а фильтрация станций — в `features/stations/filterStations.ts`.

`CarShell` является чистым map-first контейнером со слотами:

- `map` — абсолютный полноэкранный базовый слой;
- `routePanel` — компактная сводка маршрута;
- `editorPanel` — временный редактор точек А/Б или поиска рядом;
- `filterDock` — компактный нижний выбор топлива, радиуса/коридора и фильтров;
- `mapControls` — быстрый возврат к редактированию;
- `stationPanel` — управляемый drawer выбранной АЗС.

Состояние карты и UI drawer разделены: выбор станции не пересоздаёт геометрию маршрута и не запускает повторный `fitBounds`. Фокус станции передаётся типизированным prop без глобальных DOM-событий. На desktop `StationDetailsDrawer` ограничен шириной `clamp(360px, 27vw, 440px)` и закрывается кнопкой или Escape. До 767 px тот же компонент становится bottom sheet с состояниями half/expanded; карта остаётся видимой.

Удалены прежний универсальный `BottomSheet`, дублирующий `MapContainer`, старые Header/DrivePanel, скрытый список карточек, неиспользуемые сводки и зависимость `framer-motion`. Проверки `noUnusedLocals` и `noUnusedParameters` включены в основной TypeScript-конфиг frontend.

Детальные API-запросы выполняются только для выбранной АЗС. Отдельный слой Smart Advice и рекомендованное состояние маркеров удалены из интерфейса.

## Сохранённые контракты и сценарии

- режимы «Рядом» и «По маршруту»;
- GPS, ручная точка, выбор на карте, поиск и reverse geocode;
- смена точек А/Б и построение OpenRouteService route geometry;
- distance, duration, corridor/radius и топливные фильтры;
- `/api/fuel/nearby`, `/api/fuel/route-real`, `/api/geo/search`, `/api/geo/reverse`;
- localStorage, PWA и Yandex Maps;
- ручные drag/zoom без автоматического возврата карты;
- lazy details/comments/recent только для выбранной станции.

## Адаптив и производительность

Desktop, tablet и mobile используют одну DOM-архитектуру без параллельного рендера списка деталей. Маркеры остаются memoized, видимый массив станций вычисляется через `useMemo`, обработчики выбора карты — через `useCallback`, bounds/zoom — с существующим throttle. CSS использует только анимации transform/opacity/height продолжительностью 220 мс и учитывает `prefers-reduced-motion` и safe-area.
